Welcome to MyServer Backend API

This backend powers a role-based program management platform. All endpoints (except Auth) are secured and require authentication.

Authentication

  • POST /register – Create a new user account.
    Body: { "user": "name", "pwd": "password" }
  • POST /auth – Login with username & password. Returns accessToken and sets refreshToken cookie.
  • GET /refresh – Use the secure cookie to obtain a new access token.
  • GET /logout – Logout and clear the refresh token cookie.

After login, include the accessToken in request headers: Authorization: Bearer <accessToken>

Available Dashboard Endpoints

  • GET /api/dashboard/activists – List all activists (Admin only)
  • GET /api/dashboard/activist/:id – Get an activist by ID (Admin only)
  • GET /api/dashboard/program – Get all programs (Admin only)
  • POST /api/dashboard/program – Enter a new program (Editor only)
  • GET /api/dashboard/program/:id – Get program by ID (Admin/Editor)
  • PUT /api/dashboard/program/:id – Edit program by ID (Admin/Editor)
  • DELETE /api/dashboard/program/:id – Delete program by ID (Admin/Editor)

Notes for Developers

1. All dashboard endpoints are protected; only users with correct roles can access them.
2. Use JWT access tokens in Authorization header for API requests.
3. Refresh token is stored in httpOnly cookie for token renewal.
4. Admin, Editor, and User roles are supported.
